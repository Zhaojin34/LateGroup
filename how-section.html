1. Resources & Tools

Tool Name Descriptoin Rationale
HTML5 The language used to display content on the internet. We need to use this tool as it is the latest version of HTML.
CSS Paired with HTML5, CSS (Cascading Style Sheets) is used to change how the information given through HTML5 is displayed, allowing to change fonts, colours, alignment etc. In an attempt to make the website more usable, were using CSS, in alignment Nielsen’s Principles of Usability, to improve usability.
JavaScript Allows the website to be interactive. The goal is to have an account saved on the website. This account should have a virtual pet panda – which grows as tasks/events are completed.
Amazon S3 Described as a “simple storage service” it is scalable and secure. Allowing the group to upload the HTML5, CSS and JavaScript files to host safely on Amazon S3. This tool is required to submit the assignment. However, the website also needs to be hosted regardless of the assignment.

2. Collaborative Workspaces

The group will be using trello to organize the project, (https://trello.com/b/TA3pkQ73/ar-fr-330-lategroup). We will communicate on a private Microsoft Teams chat, and files will be shared on GitHub (https://github.com/patricksoliman/LateGroup)


3. Communication Expectations

The group expects to talk on Microsoft Teams every 2 days to update other members on the progress made on their respective parts of the project. Furthermore, a weekly group meeting will be held and recorded on Microsoft Teams to clarify any queries between members – in hopes that the website will meet the project plan and expectations.


4. Decision-making Processes

All decisions will be proposed by the team leader and voted/discussed within the team. The team has also agreed that a secondary vote cannot happen – this is to reduce confusion between the team, and allow the team to stay on schedule.
